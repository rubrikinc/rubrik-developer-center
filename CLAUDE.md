# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

This is the source for [developer.rubrik.com](https://developer.rubrik.com), a MkDocs-Material static site documenting the Rubrik Security Cloud (RSC) GraphQL API, SDKs, and integrations.

## Local Development

```bash
# First-time setup
./setup.sh           # creates venv and installs Python deps

# Serve locally (http://localhost:8000)
source venv/bin/activate
mkdocs serve
```

The `llms.txt` file (following llmstxt.org standard) is generated by the `mkdocs-llmstxt` plugin and available at `/llms.txt` during local dev.

## Architecture

### Content / Code Split

Documentation lives in `docs/` and code samples live in the parallel `code/` tree. MkDocs uses `pymdownx.snippets` to inline code files into markdown at build time via the `--8<--` directive:

```markdown
--8<-- "code/Rubrik-Security-Cloud-API/someFile.gql"
```

The `code/` and `docs/` directory structures mirror each other (e.g., `docs/Rubrik-Security-Cloud-API/Data-Protection/Cloud/GCP.md` pulls from `code/Rubrik-Security-Cloud-API/Data-Protection/Cloud/GCP/`).

### Code Sample Conventions

Each feature typically has three parallel sample files:
- `.gql` — raw GraphQL query/mutation
- `.sh` — shell script using `curl` with Bearer token auth
- `.ps1` — PowerShell SDK equivalent

### Schemas

GraphQL schema snapshots are stored in `docs/Rubrik-Security-Cloud-API/schemas/` with filenames in `YYYYMMDD.graphql` format (weekly cadence). The `.graphqlconfig` file points to the latest schema for VSCode GraphQL extension introspection.

### Tools (Private / Gitignored)

Scripts in `tools/` automate schema and doc maintenance. Several are gitignored (listed in `.gitignore`) because they depend on a private schema repo via `gh` CLI:

| Script | Purpose |
|--------|---------|
| `tools/generate_comprehensive_changelog.py` | Diffs consecutive schema files using `graphql-inspector` to build `Changelog.md` |
| `tools/extract_deprecated_fields.py` | Parses `@deprecated` directives from latest schema to update `Deprecations.md` |
| `tools/generate_event_reference.py` | Reads `rsc-events.csv` to generate the `Observability/Events/Event-Reference/` pages |
| `tools/generate_postman_collection.py` | Walks `code/` for `.gql` files to produce a Postman collection |
| `tools/generate_curl_examples.py` | Generates curl examples from `.gql` files |
| `tools/check_new_schemas.py` | Uses `gh` CLI to check for new schema branches in the private repo |

### Auto-Generated Content

- `docs/Rubrik-Security-Cloud-API/Changelog.md` — generated by `generate_comprehensive_changelog.py`
- `docs/Rubrik-Security-Cloud-API/Deprecations.md` — generated by `extract_deprecated_fields.py`
- `docs/Rubrik-Security-Cloud-API/Observability/Events/Event-Reference/` — generated by `generate_event_reference.py` from `rsc-events.csv`

Do not manually edit these files; regenerate them using the corresponding tool.

### Navigation

Page ordering uses the `mkdocs-awesome-pages` plugin (`.pages` files in directories). The `includes/glossary.md` file is auto-appended to every page via `pymdownx.snippets`, providing abbreviation tooltips (RSC, SLA, GraphQL, Snappable).

## Key Conventions

- The RSC API has a single endpoint: `POST /api/graphql`
- Authentication uses OAuth2 Client Credentials (service accounts) for automation, or OAuth2 Authorization Code + PKCE for user-context flows (Annapurna retrievers only)
- Access tokens are Bearer tokens in the `Authorization` header
- Schema changelog entries are categorized as Breaking, Potentially Breaking, or Non-Breaking based on `graphql-inspector` output
